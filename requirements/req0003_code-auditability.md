# REQ-0003: Notebook code is auditable without execution

## Description

The dialogue analyser notebook currently has several code quality issues that make it difficult to reason about correctness without running the notebook:

- Functions such as `label_cluster`, `run_for_k`, and several robustness helpers are defined in multiple cells across the notebook. In a live kernel the last executed cell wins, but the source alone does not make the authoritative definition clear.
- The notebook is approximately 7 MB because stored cell outputs (plots, traces, printed tables) are embedded in the `.ipynb` JSON. This makes git diffs nearly unreadable and obscures which cells are source vs cached output.
- Section numbering is inconsistent (numerals 1–8 in Part I, alphanumeric 2B–8B in Part II, Roman IV.A/B in Part IV), making navigation harder than necessary.

The requirement is that someone reading the notebook source — without running it — should be able to identify the single authoritative definition of each function, understand the logical structure, and verify the analysis workflow.

**Why this matters**: Code that cannot be audited without execution is a practical barrier to collaboration, peer review, and catching bugs like the sensitivity overwrite (REQ-0002). Jess specifically mentioned getting "into a mess on function names" as a concrete manifestation of this problem.

**Who benefits**: Neil, Jess, any future collaborator, and reviewers checking methodology.

## Acceptance Criteria

- [ ] Each helper function has exactly one definition in the notebook source (or in an imported module)
- [ ] Stored cell outputs are stripped from the `.ipynb` file committed to version control (outputs can be regenerated by running the notebook)
- [ ] Section numbering follows a single consistent scheme throughout the notebook
- [ ] A short "function index" comment cell or markdown cell lists the functions defined in each major section

## Notes

Stripping outputs from the notebook does not mean outputs are lost — the `outputs/` directory and any downloaded zip contain the results. The `.ipynb` file in version control should be a clean source artefact.
