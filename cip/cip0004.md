## Summary

Investigate and reduce the "public dialogue" artefact: extracted phrases that reflect the framing of the extraction prompt rather than the substantive content of the source paragraphs. This involves (1) adding a vocabulary frequency diagnostic to make over-represented meta-vocabulary visible, (2) revising the extraction prompt to reduce priming, and (3) optionally applying a meta-vocabulary stop-list. Implements [REQ-0004](../requirements/req0004_artefact-reduction.md). Depends on [CIP-0001](./cip0001.md) for filter drop data.

## Motivation

Jess observed: "the fact that public dialogue comes up so much in concerns and benefits makes me think it is just people talking about public dialogue." This is a plausible hypothesis and needs to be tested and addressed.

The current extraction prompt for concerns includes:

> "You are analysing a paragraph from a UK public dialogue report on science and technology..."

This preamble tells the LLM the context before showing the paragraph. A language model that is uncertain what concern to extract from an uninformative paragraph may default to generic process language ("ensuring the public dialogue is inclusive", "engaging the public in dialogue") rather than returning `NO_CONCERN`. Because "public dialogue", "dialogue participants", and "engagement" are not in `tech_words`, these phrases pass the filter and are embedded and clustered alongside genuine concerns.

The effect compounds at the cluster labelling stage: if many extracted phrases contain "public dialogue", the cluster label will naturally use that phrase, making the cluster appear substantive when it may be an artefact.

### Dependency on CIP-0001

[CIP-0001](./cip0001.md) produces `tech_filter_drops.csv`. Before revising the prompt, it is worth checking whether prompt-induced phrases are also being dropped by the filter — i.e. whether the problem is smaller than it appears. The filter drop data provides this baseline.

## Detailed Description

### Step 1: Vocabulary frequency diagnostic

Before changing anything, measure the current state. Add a post-extraction cell that computes:

```python
from collections import Counter
import re

def phrase_tokens(phrases):
    for p in phrases:
        tokens = re.findall(r'\b[a-z]{3,}\b', p.lower())
        yield from tokens
        # also yield bigrams
        for a, b in zip(tokens, tokens[1:]):
            yield f"{a} {b}"

concern_token_counts = Counter(phrase_tokens(concerns_df['concern_phrase']))
benefit_token_counts = Counter(phrase_tokens(benefits_df['benefit_phrase']))
```

Write the top-100 unigrams and bigrams to `concern_vocab_frequency.csv` and `benefit_vocab_frequency.csv`. This makes it possible to see objectively how prominent "public dialogue", "dialogue participants", etc. are.

**Proposed threshold for "over-represented":** any bigram that appears in >10% of all extracted phrases, where that bigram is a meta-vocabulary term (not a substantive concern/benefit concept), is flagged. A curated meta-vocabulary list is maintained as a constant in the notebook:

```python
META_VOCABULARY = [
    "public dialogue",
    "dialogue participants",
    "public engagement",
    "engagement process",
    "dialogue process",
    "public consultation",
    "stakeholder engagement",
]
```

### Step 2: Prompt revision options

Three prompt revision strategies, in increasing strength:

**Option A — Explicit instruction (least disruptive)**

Add to the existing prompt:

> "Do NOT use the words 'public dialogue', 'dialogue', 'engagement', 'consultation', or 'participation' in your extracted phrases. Focus on the substantive concern raised, not the process."

This preserves context for the model while directly suppressing the offending vocabulary.

**Option B — Remove corpus framing**

Replace:
> "You are analysing a paragraph from a UK public dialogue report on science and technology..."

With:
> "You are analysing a paragraph from a public report on science and technology..."

This reduces the specificity of the framing without removing all context.

**Option C — Neutral framing**

Remove corpus description entirely and provide only the extraction task:
> "Extract the main public concern expressed in the following paragraph, rephrased without technology-specific terms..."

This is the most aggressive change and may reduce extraction quality on substantive paragraphs if the model needs context to interpret ambiguous text.

**Recommended approach:** Start with Option A. If the vocabulary diagnostic after Option A still shows meta-vocabulary over-representation, proceed to Option B. Option C is a fallback.

### Step 3: Re-run diagnostic after revision

After any prompt change, re-run extraction on a sample of paragraphs (or from a checkpoint if available) and re-compute the vocabulary frequency diagnostic to compare.

### Step 4: Meta-vocabulary stop-list (optional)

If prompt revision alone is insufficient, add `META_VOCABULARY` terms to a secondary filter applied after the `tech_words` filter. Log drops to a separate `meta_vocab_drops.csv` file (parallel to `tech_filter_drops.csv` from CIP-0001).

This is a last resort because it may remove genuinely substantive phrases about participatory governance, which is a real concern in AI dialogues.

### Note on cluster labels

Even after prompt revision, existing cluster labels (in `cluster_labels.json`) may contain "dialogue" vocabulary because they were generated from the old extraction outputs. After re-running extraction, cluster labels should be regenerated. This is expected behaviour, not a bug.

2. **Inspect diagnostic output** against current `extracted_concerns.csv` / `extracted_benefits.csv` to quantify the artefact

6. **Re-run extraction on a sample** (50–100 paragraphs) and re-run vocabulary diagnostic to assess improvement

9. **Regenerate cluster labels** after re-extraction

## Backward Compatibility

- Prompt changes will produce different extracted phrases, which will produce different clusters and cluster labels. This is expected and intentional — the goal is to improve the analysis.
- If there are any figures or analyses in draft paper sections that reference specific cluster labels or phrase examples from the current run, those will need updating after re-extraction.
- The `extracted_concerns.csv` and `extracted_benefits.csv` from the current run should be archived (e.g. as `extracted_concerns_v12b.csv`) before overwriting.

## Testing Strategy

- Compare vocabulary frequency diagnostics before and after prompt revision — META_VOCABULARY terms should appear less frequently in the post-revision extraction
- Manually review 20 extracted phrases before and after to assess qualitative improvement
- Check that the revised prompt does not suppress genuinely substantive phrases (e.g. concerns about AI governance, which legitimately involve "public" vocabulary)
