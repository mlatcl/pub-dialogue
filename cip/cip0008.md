## Summary

The notebook has a k-sensitivity analysis (how do results vary with number of
clusters?) but no prompt-sensitivity analysis (how much do results depend on the
exact wording of the extraction prompt?). Given that a single prompt word change
can materially shift LLM outputs, and given that the paper presents findings as
representative of the data rather than artefacts of a specific prompt, this gap
needs to be addressed before submission.

## Motivation

From the review:

> There is no sensitivity analysis for the extraction prompt itself. With LLM
> extraction it is well known that small phrasings changes can produce very
> different sets. If the results are highly sensitive to prompt wording, the
> paper's claims weaken considerably. At minimum, two or three prompt variants
> should be tested and inter-prompt agreement reported.

The paper currently presents clusters as if they are a stable, objective
summary of participant concerns. A prompt sensitivity analysis is needed to
justify this framing.

## Detailed Description

### What to test

Test at least 3 prompt variants per kind (concern, benefit):

**Variant A (current):** The prompt in `EXTRACTION_PROMPT` /
`BENEFIT_EXTRACTION_PROMPT` as currently implemented.

**Variant B (paraphrase):** Semantically equivalent but differently worded.
For concerns, e.g. replace "Extract distinct concerns or worries…" with
"Identify the specific anxieties, objections, or reservations expressed…".

**Variant C (minimal):** A stripped-down prompt with fewer constraints and
no negative examples.

### What to measure

For each pair of prompt variants:
- **Phrase-level agreement**: for each chunk, what fraction of extracted phrases
  are semantically equivalent across variants? Use embedding cosine similarity
  ≥ 0.85 as "equivalent".
- **Yield agreement**: does Variant B flag the same chunks as yielding ≥1
  concern/benefit?
- **Cluster stability**: do the same 5/7/10 cluster solutions produce recognisably
  the same themes across variants (label similarity, member overlap)?

### Acceptable thresholds

A reasonable standard: ≥70% phrase-level agreement across variants, ≥85%
yield agreement, top-5 clusters stable in name and composition. If below
these thresholds, the paper should note that specific claims are prompt-dependent
and present results as indicative rather than definitive.

### Implementation

Add a `run_prompt_sensitivity` function to `dialogue_utils.py`:

```python
def run_prompt_sensitivity(
    chunks: pd.DataFrame,
    kind: str,
    prompts: Dict[str, str],  # {variant_name: system_prompt}
    client,
    model: str,
    sample_n: int = 200,
    output_folder: Path,
) -> pd.DataFrame:
    """Run extraction with multiple prompt variants on a sample of chunks."""
```

Run on a stratified sample of 200 chunks (balanced across technologies) rather
than the full corpus, to keep cost manageable.
