# CIP-0009: Temporal analysis methodology and document weighting

## Summary

The temporal trend charts in the notebook plot raw counts of concern/benefit
phrases per cluster per year. This confounds two separate signals: (1) changes
in the number of dialogues conducted per year and (2) genuine changes in what
participants said. The 2023–2024 surge in "Benefits of innovation" is almost
entirely driven by more dialogues being conducted in those years, not by
participants becoming more positive. Additionally, documents are weighted
implicitly by their number of paragraphs (longer documents dominate), which
further confounds the signal. CIP-0009 proposes a methodology revision before
the temporal analysis is included in the paper.

## Motivation

From the review:

> The temporal trend charts plot raw phrase counts per year. There is no
> normalisation by the number of dialogues per year, number of source
> paragraphs, or number of documents. There is also an implicit document
> weighting: a 120-page report contributes 3× as many phrases as a 40-page
> report, regardless of the number of distinct concerns it actually raises.

**Issue 4 (temporal analysis)**:
- Year field parsed from filename is unreliable — "2018 Nuclear Energy" and
  "2018 Genome Editing" are both 2018 but the field is populated from a regex
  that may fail on unusual filenames.
- Plot labels mislabel which year corresponds to which cluster prominence.

**Issue 10 (document weighting)**:
- Long documents dominate phrase counts regardless of thematic diversity.
- A document-level aggregation (e.g. count cluster members per document rather
  than per phrase) would give each document equal weight.

## Detailed Description

### Approach A: Normalise by dialogue count (minimum fix)

Replace raw counts with fraction of dialogues in a given year that mention
cluster C. This is robust to year-on-year variation in corpus size.

### Approach B: Document-level weighting (fuller fix)

Before temporal analysis, aggregate phrases to document level: for each
(document, cluster), record whether the cluster is mentioned (binary) or a
count normalised by document length. Temporal trends then show % of documents
per year that raise each concern.

**Decision required:** Which approach to use in the paper?

Both are straightforward to implement. Approach A is the minimal fix; Approach
B is more defensible methodologically and should be the paper standard.

### Approach C: Remove temporal analysis (deferred)

If time is short, remove or explicitly caveat the temporal analysis as
"exploratory, unnormalised" in the paper rather than fixing it.

### Year field validation

Add a validation step that:
1. Compares the regex-extracted year against a `year` column in the metadata
   Excel (if present).
2. Reports any mismatches.
3. Flags documents with no valid year.
