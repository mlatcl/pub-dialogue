# Task: Test validation playbook with Jess and integrate into notebook export

> **Note**: Backlog tasks are DOING the work defined in CIPs (HOW).  
> Use `related_cips` to link to CIPs. Don't link directly to requirements (bottom-up pattern).

## Description

Jess follows the draft `validation_playbook.md` against the current set of outputs in `outputs/`. Any step that is unclear, references a file that does not exist, or produces ambiguous results is noted. The playbook is then revised and, once satisfactory, integrated into the notebook export so it is included in every export pack.

## Acceptance Criteria

- [ ] Jess has followed all four activities in the playbook using the current `outputs/` files and noted any issues
- [ ] All unclear or broken steps have been revised in `validation_playbook.md`
- [ ] Optional Python snippets have been added in an appendix for steps that are tedious without code (e.g. random sampling with `pandas`, cross-technology filtering)
- [ ] The notebook's Part V (export) cell includes `validation_playbook.md` in the zip archive
- [ ] A "validation summary" block is added to the `evidence_pack_paragraphs.html` generation: a header section showing total clusters, total phrases, and record counts — pre-populating the numbers needed for Activity 4
- [ ] After integration, a test export is run and the downloaded zip is verified to contain `validation_playbook.md`

## Implementation Notes

The usability test should be done with the actual `outputs/` files, not a synthetic dataset. Jess should treat the playbook as if seeing it for the first time — the goal is to find steps that assume knowledge not documented in the playbook.

The "validation summary" block in `evidence_pack_paragraphs.html` should appear at the top of the HTML, before the cluster-by-cluster content. It can be generated by the same cell that produces the HTML, using data already available (cluster count from `cluster_summary.csv`, phrase count from `extracted_concerns.csv`, etc.).
